## Managing user attributes

Attributes such as `name`, `email`, `phone_number` help you identify individual users. The Amplify JavaScript library contains 
a set of APIs that can help you manage attributes while the user is authenticated.

### Pass user attributes during sign up

Amazon Cognito has a set of default standard attributes. You can find a list of all the standard attributes [here](https://docs.aws.amazon.com/cognito/latest/developerguide/user-pool-settings-attributes.html#cognito-user-pools-standard-attributes)

In addition, Cognito allows to setup up to 50 custom attributes in the user pool. Documentation on custom attributes can be found [here](https://docs.aws.amazon.com/cognito/latest/developerguide/user-pool-settings-attributes.html#user-pool-settings-custom-attributes).

You can pass them in the `userAttributes` key of the `signUp` API:

```javascript
signUp({
	username: 'jdoe',
	password: 'mysecurerandompassword#123',
	options: {
		userAttributes: {
			email: 'me@domain.com',
			phone_number: '+12128601234',
			given_name: 'Jane',
			family_name: 'Doe',
			nickname: 'Jane',
			'custom:attribute_name': 'attribute_value',
		},
	},
});
```

Some attributes require confirmation for the attribute update to complete. If the attribute need to be confirmed, the result of the above API will be `CONFIRM_ATTRIBUTE_WITH_CODE`. A confirmation code will be sent to the delivery medium mentioned in the delivery details.
